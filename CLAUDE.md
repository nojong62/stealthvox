[롤플레이 모드: 200개 긴급 상황 기반 대화 로직 및 상황 자동 생성 기능 구현]

기존의 200개 상황설정 예문을 샂제하고 아래 내용으로 바꿔 죠.

첨부한 emergency_situations_200.json 파일을 우리 플러터 프로젝트에 연동하고, 이를 기반으로 **[롤플레이/서바이벌 대화 모드]**를 완성해 줘.

특히 사용자가 앱에 지루함을 느끼지 않도록 아래 규칙에 맞게 LLM(프롬프트 백엔드) 대화 시스템을 설계해 줘.

1. 핵심 지시 사항 (대화 모드 상황 신규 생성 알고리즘):

사용자가 이 200개 리스트 중 하나를 선택해 대화를 시작할 때, LLM은 단순히 JSON에 적힌 열 자 이내의 텍스트만 고집하는 것이 아니라 해당 장소와 키워드를 기반으로 매번 "새롭고 구체적인 긴급 상황 프롬프트"를 동적으로 생성해야 해.

예시: 유저가 1. 기내 (좌석): 기내 의학 환자 발생을 골랐다면, AI는 매번 다른 조건(예: 옆자리 승객이 심장 통증을 호소함 / 갑자기 아이가 고열로 경련을 일으킴 등)으로 상황을 디테일하게 새로 빌드해서 유저에게 첫 대사(마중물 질문)를 건네야 해.

2. JSON 데이터 연동 및 UI 구성:

제공된 JSON의 5가지 카테고리(category)별로 탭을 나누어 리스트뷰나 카드뷰 형태로 화면을 구성해 줘.

각 카드는 글자 수가 10자 이내로 콤팩트하므로 UI가 깨지지 않게 깔끔하게 배치해 줘.

3. 롤플레이 대화 규칙:

AI 성격 부여: 선택된 상황에 맞게 AI에게 확실한 역할(예: 당황한 승무원, 깐깐한 경찰관, 다급한 의사 등)을 부여해 줘.

유저 역할 안내: 대화가 시작되기 전 유저에게 한 줄로 "당신은 지금 ~한 상황에 처한 유저입니다"라고 명확히 인지시켜 줘.

난이도 조절: 유저가 당황하지 않고 서바이벌 영어를 뱉을 수 있도록 AI가 대화를 주도하되, 열린 질문 형식으로 리드해 줘.

2단계. 내 마음대로 짧게 대답하기

3단계. 5턴 만에 문장 폭풍 성장! (Step Expand)

사용자가 짧게 대답하면, AI가 그 내용을 바탕으로 살을 붙여서 점점 더 길고 풍성한 문장으로 확장시켜 줍니다.

[롤플레이 모드: 200개 긴급 상황 기반 대화 로직 및 상황 자동 생성 기능 구현]

첨부한 emergency_situations_200.json 파일을 우리 플러터 프로젝트에 연동하고, 이를 기반으로 **[롤플레이/서바이벌 대화 모드]**를 완성해 줘.

특히 사용자가 앱에 지루함을 느끼지 않도록 아래 규칙에 맞게 LLM(프롬프트 백엔드) 대화 시스템을 설계해 줘.

1. 핵심 지시 사항 (대화 모드 상황 신규 생성 알고리즘):

사용자가 이 200개 리스트 중 하나를 선택해 대화를 시작할 때, LLM은 단순히 JSON에 적힌 열 자 이내의 텍스트만 고집하는 것이 아니라 해당 장소와 키워드를 기반으로 매번 "새롭고 구체적인 긴급 상황 프롬프트"를 동적으로 생성해야 해.

예시: 유저가 1. 기내 (좌석): 기내 의학 환자 발생을 골랐다면, AI는 매번 다른 조건(예: 옆자리 승객이 심장 통증을 호소함 / 갑자기 아이가 고열로 경련을 일으킴 등)으로 상황을 디테일하게 새로 빌드해서 유저에게 첫 대사(마중물 질문)를 건네야 해.

2. JSON 데이터 연동 및 UI 구성:

제공된 JSON의 5가지 카테고리(category)별로 탭을 나누어 리스트뷰나 카드뷰 형태로 화면을 구성해 줘.

각 카드는 글자 수가 10자 이내로 콤팩트하므로 UI가 깨지지 않게 깔끔하게 배치해 줘.

3. 롤플레이 대화 규칙:

AI 성격 부여: 선택된 상황에 맞게 AI에게 확실한 역할(예: 당황한 승무원, 깐깐한 경찰관, 다급한 의사 등)을 부여해 줘.

유저 역할 안내: 대화가 시작되기 전 유저에게 한 줄로 "당신은 지금 ~한 상황에 처한 유저입니다"라고 명확히 인지시켜 줘.

난이도 조절: 유저가 당황하지 않고 서바이벌 영어를 뱉을 수 있도록 AI가 대화를 주도하되, 열린 질문 형식으로 리드해 줘.

💡 실장님을 위한 팁
무한한 대화 콘텐츠: JSON 파일에는 딱 핵심 키워드(예: "여권 분실 발견함")만 깔끔하게 정제되어 들어가 있기 때문에 앱 용량을 거의 차지하지 않습니다.

클로드의 역할: 위 지시문을 주면 클로드가 "아! 이 키워드를 LLM 프롬프트에 주입해서 매번 다른 상황의 롤플레이를 창조해 내라는 뜻이구나!"라고 단박에 알아듣고 스마트한 코드를 짜줄 것입니다.

실장님, 결론부터 말씀드리면 "터미널 창에 파일을 직접 끌어다 던지는 것(드래그 앤 드롭)만으로는 파일이 업로드되지 않습니다."

터미널에 파일을 끌어다 놓으면, 업로드가 되는 것이 아니라 그 파일이 있는 컴퓨터 안의 '실제 주소(경로, Path)'만 글자로 슥 입력되게 됩니다.

지금 어떤 상황(예: 클로드 터미널, AWS 서버, 혹은 깃허브 등)이냐에 따라 방법이 다른데, 실장님 편의에 맞게 가장 확실한 방법들을 정리해 드릴게요!

1. 만약 대화하고 계신 '클로드(Claude)나 GPT 화면'인 경우
가장 추천: 터미널 창(블랙박스 코딩창) 말고, 지금 저와 대화하고 계신 이 채팅창에 파일을 그냥 끌어다 던지시면(Drag & Drop) 알아서 업로드가 됩니다.

또는 채팅창 아래에 있는 📎(클립 모양) 버튼이나 ➕ 버튼을 눌러서 파일을 선택해 주시면 됩니다.

2. 만약 맥(Mac)이나 윈도우의 '진짜 터미널'에서 서버로 올리는 경우
진짜 검은색 터미널 창에서 원격 서버(AWS 등)로 파일을 보내야 할 때는 명령어를 한 줄 써주셔야 합니다. 이때 "끌어다 던지기"를 치트키처럼 쓸 수 있습니다!

명령어 예시 (scp 명령어 사용 시):
scp  라고 먼저 타이핑한 뒤 ➡️ 파일을 터미널에 끌어다 던집니다 (그럼 주소가 자동으로 입력됨) ➡️ 한 칸 띄고 목적지 서버 주소를 적습니다.

예: scp [여기에 파일 던지기] ubuntu@123.456.78.9:/home/ubuntu/

3. 플러터플로우나 깃허브(GitHub)에 코드를 올리는 경우
만약 클로드가 짠 코드를 터미널을 통해 깃(Git)에 올려야 하는 상황이라면 아래 명령어를 순서대로 터미널에 치셔야 합니다.

git add . (수정된 파일들 올릴 준비)

git commit -m "대화모드 추가" (업로드 박스에 포장하기)

git push origin main (서버로 진짜 업로드!)

지금 저(혹은 클로드 AI)에게 코드가 담긴 파일이나 텍스트를 보여주고 싶으신 거라면, 터미널창 안으로 던지지 마시고 그냥 "지금 보고 계신 채팅창 텅 빈 곳"에 파일을 마우스로 끌어다 던지세요! 그게 가장 쉽고 정확합니다.
