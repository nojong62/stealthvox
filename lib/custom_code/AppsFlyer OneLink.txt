준비해야 할 AppsFlyer OneLink   핵심 자료
1. OneLink 템플릿 ID (Template ID)

AppsFlyer 관리자 화면의 Engage -> OneLink Management에서 생성한 템플릿의 고유 ID입니다.

안드로이드와 iOS 통합 링크를 만들기 위한 '뿌리' 주소입니다. (예: a1b2.onelink.me)

2. 초대자 식별용 커스텀 파라미터 설계

링크 뒤에 붙일 데이터의 이름을 정해야 합니다. 보통 다음과 같이 구성합니다.

deep_link_value: 실행될 화면 이름 (예: chat_room)

inviter_id: 초대한 회원의 고유 ID (예: user_1234)

room_id: 진입할 채팅방의 고유 ID

3. 앱 내 SDK 연동 가이드 (Dev Guide)

FlutterFlow를 사용 중이시라면, AppsFlyer SDK가 앱 안에 심어져 있어야 합니다.

특히 onDeepLinking 콜백 함수를 통해 링크에 담긴 inviter_id와 room_id 값을 앱이 읽어낼 수 있도록 설정하는 가이드 문서가 필요합니다.

4. 리어트리뷰션(Re-attribution) 설정 값

이미 앱이 설치된 기존 회원들끼리도 링크를 눌렀을 때 작동해야 하므로, AppsFlyer 대시보드에서 Re-engagement 기능을 활성화해두어야 합니다.

🛠️ 구현 로직 (개발자에게 전달할 내용)
링크를 타고 들어왔을 때 앱은 다음과 같이 작동해야 합니다:

링크 클릭: 사용자가 https://yourbrand.onelink.me/abc?inviter_id=123&room_id=99 클릭.

데이터 수신: AppsFlyer SDK가 앱 실행 시 inviter_id=123과 room_id=99를 잡아냄.

자동 진입: 앱 로직에서 이 값을 확인하자마자 로그인 세션을 체크하고, 해당 room_id 채팅방으로 화면을 즉시 이동(Navigate)시킴.

추천코드 자동 적용: 필요하다면 inviter_id를 백엔드로 보내서 "추천인 입력" 과정을 백그라운드에서 자동으로 처리.
