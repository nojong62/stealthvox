첫 결제 테스트 시 콘솔/로그에 다음 메시지가 뜰 가능성이 매우 높습니다:
[cloud_firestore/failed-precondition] The query requires an index. 
You can create it here: 
https://console.firebase.google.com/v1/r/project/.../indexes?create_composite=...
이건 에러가 아니라 Firebase가 "복합 인덱스 만들어 줄게~ 링크 누르세요" 라고 알려주는 거예요.
대처법:
위 메시지의 링크를 그대로 클릭
Firebase Console로 이동
"Create Index" 또는 "색인 만들기" 버튼 클릭
1~2분 기다리면 인덱스 생성 완료
그 후로는 결제 정상 작동

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    
    // chat_history는 본인만
    match /users/{uid}/chat_history/{room} {
      allow read, write: if request.auth.uid == uid;
      
      match /messages/{msg} {
        allow read, write: if request.auth.uid == uid;
      }
    }
    
    // sessions는 본인만
    match /users/{uid}/sessions/{session} {
      allow read, write: if request.auth.uid == uid;
      allow delete: if false;
    }
    
    // purchases는 본인만 (이미 OK)
    match /users/{uid}/purchases/{doc} {
      allow read, write: if request.auth.uid == uid;
    }
    
    // users 본문은 본인만
    match /users/{uid} {
      allow read, write: if request.auth.uid == uid;
    }
    
    // rooms는 듀오 모드 게스트가 들어올 수 있어야 하니 일단 인증만 요구
    match /rooms/{room} {
      allow read, write: if request.auth != null;
      
      match /messages/{msg} {
        allow read, write: if request.auth != null;
      }
    }
    
    // 다른 모든 것은 거부
    match /{document=**} {
      allow read, write: if false;
    }
  }
}
