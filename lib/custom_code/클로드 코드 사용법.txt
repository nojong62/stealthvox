  1. "찾아서 고쳐라" vs "새로 만들어라"                                                                                 
  ┌──────────────────────────┬─────────────────────────────────┬─────────────┐                                            │           방식           │            토큰 흐름            │ 상대적 비용 │
  ├──────────────────────────┼─────────────────────────────────┼─────────────┤
  │ Edit 도구 (찾아서 고침)  │ 파일 읽기 → 변경된 diff만 전송  │ 낮음        │
  ├──────────────────────────┼─────────────────────────────────┼─────────────┤
  │ Write 도구 (전체 재작성) │ 파일 읽기 → 전체 파일 내용 출력 │ 높음        │
  └──────────────────────────┴─────────────────────────────────┴─────────────┘

  파일이 클수록 차이가 커집니다. 1000줄 파일에서 한 줄만 바꿀 때, Edit은 그 한 줄 주변만 처리하지만 Write는 1000줄
  전체를 출력 토큰으로 소비합니다.

  2. 추상적 설명 vs 구체적 설명

  ┌──────────────────────────────────────────────┬─────────────────────────────────────────────────────┬─────────────┐
  │                     방식                     │                      토큰 흐름                      │ 상대적 비용 │
  ├──────────────────────────────────────────────┼─────────────────────────────────────────────────────┼─────────────┤
  │ 추상적 ("뭔가 버그 있어")                    │ 파일 전체 탐색 → Grep 여러 번 → Read 여러 번 → 수정 │ 높음       │
  ├──────────────────────────────────────────────┼─────────────────────────────────────────────────────┼─────────────┤
  │ 구체적 ("login.js 45번째 줄 null 체크 누락") │ 해당 위치만 Read → 즉시 수정                        │ 낮음        │
  └──────────────────────────────────────────────┴─────────────────────────────────────────────────────┴─────────────┘

  추상적일수록 탐색 도구 호출 횟수가 늘어나고, 각 호출마다 컨텍스트에 결과가 쌓여서 입력 토큰이 누적됩니다.

  ---
  실질적 절약 팁

  - 파일 경로 + 줄 번호 + 문제 설명을 같이 주면 가장 저렴합니다.
  - 큰 파일을 통째로 다시 써달라고 하면 비쌉니다. "이 부분만 고쳐"가 훨씬 효율적입니다.
  - 여러 파일을 동시에 수정할 때는 한 번에 요청하면 컨텍스트를 재활용해서 절약됩니다.

  1. Claude Code에서 모델 지정
                                                                                                                          가능합니다. 몇 가지 방법이 있습니다:

  또는 /config에서 기본 모델을 설정할 수도 있습니다.

  현재 이 세션은 claude-sonnet-4-6으로 실행 중입니다 (시스템 정보 기준).

  2. 연속 질문 vs 새 세션

  새 세션이 더 저렴합니다, 단 조건이 있습니다:

  ┌───────────────────────────────────┬────────────────────────────────────────────────┐
  │               상황                │                      추천                      │
  ├───────────────────────────────────┼────────────────────────────────────────────────┤
  │ 이전 맥락이 필요 없는 독립적 질문 │ 새 세션 — 누적된 컨텍스트 없이 시작            │
  ├───────────────────────────────────┼────────────────────────────────────────────────┤
  │ 이전 내용을 참조해야 하는 질문    │ 연속 세션 — 다시 설명하는 비용이 더 클 수 있음 │
  └───────────────────────────────────┴────────────────────────────────────────────────┘

  이유: 연속 대화에서는 매 요청마다 이전 대화 전체가 입력 토큰에 포함됩니다. 대화가 길어질수록 누적 비용이 증가합니다.

  다만 Claude Code는 프롬프트 캐싱을 사용하므로, 같은 세션 내 반복되는 컨텍스트는 할인된 캐시 토큰으로 처리됩니다.
  완전히 새로운 질문이라면 새 세션이 경제적입니다.

  1. Claude Code는 PowerShell에서 실행하나?
                                                                                                                          Claude Code는 터미널 CLI 도구입니다. PowerShell, bash, cmd 등 어떤 터미널에서도 실행 가능합니다. 현재 이 세션은 bash
  환경에서 실행 중이고, PowerShell은 Claude Code 내부에서 명령 실행할 때 사용하는 도구 중 하나입니다.                   
  터미널(PowerShell/bash/cmd)
      └─ claude 명령어로 Claude Code 실행
             └─ 내부에서 Bash/PowerShell 도구 호출 가능

  3. 웹(claude.ai)에서 받아 수동 적용 vs Claude Code에서 직접 수정

  ┌─────────────┬────────────────────────────────────────────┬─────────────────────────┐
  │    방식     │                 토큰 효율                  │        실제 편의        │
  ├─────────────┼────────────────────────────────────────────┼─────────────────────────┤
  │ 웹          │ 관련 코드만 붙여넣으면 입력 토큰 절약 가능 │ 수동으로 복붙·적용 필요 │
  ├─────────────┼────────────────────────────────────────────┼─────────────────────────┤
  │ Claude Code │ 파일 직접 읽고 Edit 도구로 diff만 처리     │ 자동 적용, 오타 없음    │
  └─────────────┴────────────────────────────────────────────┴─────────────────────────┘

  결론: 토큰 비용만 보면 비슷하거나 Claude Code가 유리합니다.

  - 웹은 파일 전체를 붙여넣으면 오히려 비쌀 수 있고, Claude Code는 필요한 부분만 Read → Edit으로 처리
  - 다만 웹에서 핵심 코드 조각만 정확히 잘라서 붙여넣으면 가장 저렴할 수도 있음
  - 실수 위험까지 고려하면 Claude Code가 확실히 유리 — 수동 복붙 실수가 없고, 여러 파일 동시 수정도 가능

* Tempering…                                                                                                           ●T모델 비용 차이 — 웹 vs Claude Code

  핵심: 과금 구조 자체가 다릅니다                                                                                       
  ┌────────────────┬───────────────────────────────────┬───────────────────────────────────────┐                          │                │          웹 (claude.ai)           │       Claude Code (API키 사용)        │
  ├────────────────┼───────────────────────────────────┼───────────────────────────────────────┤
  │ 과금 방식      │ 월정액 구독 (Pro $20/월 등)       │ 토큰당 과금                           │
  ├────────────────┼───────────────────────────────────┼───────────────────────────────────────┤
  │ 모델 전환 비용 │ 사용량 한도(limit)를 더 빨리 소모 │ Opus 4.7 = Sonnet 4.6의 5배 직접 청구 │
  ├────────────────┼───────────────────────────────────┼───────────────────────────────────────┤
  │ 비용 가시성    │ 얼마 썼는지 직접 보이지 않음      │ 토큰 수·금액 명확히 집계됨            │
  └────────────────┴───────────────────────────────────┴───────────────────────────────────────┘

  ---
  실질적 차이

  웹 구독:
  - Opus를 쓰면 같은 작업에 limit을 더 빨리 소진
  - 한도 초과 시 느려지거나 차단됨
  - 하지만 한도 내라면 추가 금액 없음

  Claude Code (API키):
  - 5배 비용 차이가 실제 청구서에 그대로 반영
  - 파일 탐색·읽기 등 도구 호출도 모두 토큰으로 과금
  - 대신 사용량 제한 없음 (돈만 내면 무제한)

  ---
  결론

  웹은 "한도 소진 속도"의 차이, Claude Code(API)는 "실제 금액"의 차이입니다. 비용이 직접 보이고 아프게 느껴지는 건 API
  방식입니다. 구독 플랜 안에서 Claude Code를 쓴다면 웹과 동일하게 한도 기반으로 동작합니다.

  ---
  사용량·시간 기준 시나리오

  가벼운 사용 (하루 5~10회 질문):
  - 웹 Pro가 압도적으로 유리 — $20/월 고정, 한도 여유 있음
  - Claude Code API는 과금이 누적되어 금방 $20 초과 가능

  중간 사용 (파일 수정, 코드 작업 1~2시간/일):
  - 웹: 한도 소진 위험, Opus → Sonnet 강제 전환 발생
  - Claude Code API: 시간 절약되지만 월 $50~100 수준 예상

  집중 작업 (하루 수십 파일, 긴 세션):
  - 웹: 한도 초과로 사실상 Opus 지속 사용 불가
  - Claude Code API: 비용 높지만 중단 없이 작업 가능

  ---
  결론

  사용량 적음  →  웹 Pro 유리 (고정 비용)
  사용량 많음  →  웹은 한도 때문에 오히려 불편
  시간 가치 높음  →  Claude Code가 자동화로 실질 절약

  현실적 추천: Opus 4.7을 자주 쓴다면 웹 Pro + Sonnet 4.6으로 Claude Code 조합이 비용·성능 균형이 가장 좋습니다. Opus는
  꼭 필요한 복잡한 작업에만 웹에서 선택적으로 사용하는 방식이 경제적입니다.
