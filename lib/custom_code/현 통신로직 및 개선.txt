 ② AI 텍스트 실시간 표시 — 무엇이 바뀌나?

 현재 상황 (1583번 라인)
```dart
if (mounted && !_ttsQueueManager.aiPaused)   // ← 이 조건이 막고 있음
    setState(() = _localMessages[aiIndex]['target'] = aiTargetText);
```

GPT가 영어 글자를 토큰 단위로 흘려보내는데, `aiPaused=true`인 동안에는 화면 업데이트를 막아둡니다. 그러다 PIPE07(유저 TTS 끝난 시점)에서 `aiPaused=false`로 바꾸면서 글자를 한 방에 표시합니다.

 사용자가 보는 화면 (현재)
```
[유저 말 끝남]
   ↓ (잠시 침묵)
   ↓ (잠시 침묵)
[AI 글자 와장창 등장 + 소리 동시 시작]
```

 변경 후 (조건 제거)
```dart
if (mounted)   // ← aiPaused 체크 빼버림
    setState(() = _localMessages[aiIndex]['target'] = aiTargetText);
```

 사용자가 보는 화면 (변경 후)
```
[유저 말 끝나기 전]
   ↓ AI 글자 똑똑똑 흘러나옴 (아직 소리 X)
   ↓ AI 글자 거의 다 나옴
[유저 소리 끝]
[AI 소리 시작 — 글자는 이미 다 보임]
```

핵심 효과: AI가 "생각하고 있다"는 시각적 피드백이 생겨서 침묵 시간이 짧게 느껴짐. 글자가 미리 보이니까 사용자가 마음의 준비도 됨.

소리는 그대로: AI 소리는 PIPE07까지 대기. 청크 큐 로직(Box 7)은 건드리지 않음 → 안전.

 ✅ 잘 된 부분
 Step Expand `utterance_end_ms=1000ms` — 단답형 유도 앱 특성에 맞게 이미 최적화됨.

 ⚠️ 속도 개선 여지 있는 구간 3곳

① Clone/Roleplay `utterance_end_ms=1200ms` → 900ms 시도 가능
 현재 1.2초 침묵 후 턴 종료. 대화 흐름에서 1.2초는 꽤 긺.
 900ms로 줄이면 체감 반응속도 확 빨라짐.
 리스크: 말하다 잠깐 숨 쉬면 조기 종료. → 테스트 필수.

② TTS speed `1.0` → `1.1` 시도 가능
 AI 응답 읽는 속도 10% 업. 대화 템포가 자연스럽게 빨라짐.
 언어학습 앱이라 너무 빠르면 역효과 — Clone/Roleplay만 적용 권장.
 Step Expand는 학습자가 따라해야 하니 1.0 유지.

③ GPT streaming timeout Clone `45s` → `30s` 통일
 45초는 너무 관대함. 실제로 45초 걸리면 사용자는 이미 앱 껐음.
 30s로 줄여도 정상 응답은 다 잡힘. 에러 복구만 빨라짐.



 🔴 건드리면 안 되는 구간
 `endpointing=700ms` — 이미 최소값에 가까움. 줄이면 문장 중간에 끊김.
 Step Expand `utterance_end_ms=1000ms` — 이미 최적. 더 줄이면 단어 자르기 사고.
 Duo temperature=0.2 — 통역 정확도용, 올리면 오역 발생.
 Box 7 — 지시사항대로 금지.
